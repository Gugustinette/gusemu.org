# Glossary

Here are some common terms and concepts related to emulation that you may encounter as you explore the world of emulation:

- <span id="bios"></span>**BIOS** (Basic Input/Output System): A firmware that initializes hardware during the booting process of a console. Some emulators require the original BIOS to function properly.
- <span id="core"></span>**Core**: An emulation module that implements the features and behavior of a specific console or system, which can be used within a frontend like RetroArch through the Libretro API.
- <span id="driver"></span>**Driver**: Software that allows the operating system to communicate with hardware devices. Some emulators may require specific drivers to function correctly.
- <span id="dump"></span>**Dump**: The end product of the ROM dumping process, which is either a ROM file (for cartridges) or an ISO file (for discs).
- <span id="emulation"></span>**Emulation**: The process of mimicking the behavior of one system on another.
- <span id="emulator"></span>**Emulator**: A software program that replicates the hardware and software of the original system, allowing you to run programs or games designed for that system on a different platform.
- <span id="firmware"></span>**Firmware**: Software that is permanently stored in a hardware device, to control its functions. Some emulators may require the original firmware to function properly.
- <span id="frontend"></span>**Frontend**: A software interface that allows users to interact with multiple emulation cores, or multiple emulators, in a unified way.
- <span id="iso"></span>**ISO** (International Organization for Standardization): A file that contains a digital copy of an optical disc, such as a CD or DVD, used for consoles that relied on discs.
- <span id="libretro"></span>**Libretro**: A standardized API for emulation cores, which allows different emulators to be used within a single frontend, such as RetroArch.
- <span id="open-source"></span>**Open source**: A type of software license that allows anyone to view, modify, and distribute the source code of a program.
- <span id="patch"></span>**Patch**: A file that contains the differences between an original ROM and a modified ROM, which can be applied to the original ROM to create the modified version without needing to distribute the entire modified ROM.
- <span id="rom"></span>**ROM** (Read-Only Memory): A file that contains the data from a video game, typically dumped from physical cartridges. Can also refer to ISO files for disc-based games.
- <span id="rom-dumping"></span>**ROM dumping**: The process of extracting the data from a physical game cartridge or disc and creating a digital copy of it, which can then be used with emulators.
- <span id="rom-hacking"></span>**ROM hacking**: The process of modifying a video game ROM to change its content.
- <span id="translation-patch"></span>**Translation patch**: A sub-genre of ROM hacks, dedicated to translating a game that was never released in certain regions, or to fix translation errors in an existing release.
